k8s1.6.6集群部署(开启TLS安全认证)（一）

发表于 2017-06-28

前面我通过yum已经部署过k8s的集群，不过没有开启集群的TLS安全认证，而且yum安装k8s官方源的最高版本号只有1.5.2，本系列开始通过二进制来部署k8s的1.6.6版本并开启安全认证。

kubernetes提供了多种安全认证方式，可以使用TLS双向认证，也可以使用token或者用户名密码的单向认证。

集群规划

IP 系统 角色 应用
172.16.110.108 centos 7.3 master node kube-apiserver kube-controller- manager kube-scheduler kubelet kube-proxy etcd flannel
172.16.110.105 centos 7.3 node kubelet kube-proxy etcd flannel
172.16.110.107 centos 7.3 node kubelet kube-proxy etcd flannel

说明：

kube-apiserver将提取CN作为客户端的用户名，这里是kubernetes-admin，将提取O作为用户所属的组，这里是system:master。

kube-apiserver预定义了一些 RBAC使用的ClusterRoleBindings，例如 cluster-admin将组system:masters与 ClusterRole cluster-admin绑定，而cluster-admin拥有访问kube-apiserver的所有权限，因此kubernetes-admin这个用户将作为集群的超级管理员。
